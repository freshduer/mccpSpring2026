LONGER

Scaling Up Long Sequence Modeling in Industrial Recommenders

Chai, Ren, Xiao, Yang et al. (ByteDance) · RecSys '25

Wenjun Yu  ·  2026.3.2

Motivation

  • User behavior history (clicks, purchases, watches) is the primary signal for recommendation quality.
  • Longer history → richer signals → consistently higher accuracy.
  • But standard Transformer scales as O(N²) — infeasible beyond N ~ 500 in production.
  • Two-stage solutions (SIM, ETA, TWIN): pre-filter to top-K then model — creates an irreversible information bottleneck.
  • Goal: end-to-end modeling of the full behavior sequence at industrial scale.

Core Idea

Challenge: O(N²) attention — at N=10k, costs 400× more than N=500.

  • ① OuterTrans (Global tokens)
    G learnable tokens aggregate full history via cross-attention; complexity O(N²) → O(G·N), G≪N.
  • ② InnerTrans (Token Merge)
    Adjacent tokens with high cosine similarity merged before attention; effective N cut ~50%.
  • ③ Hybrid Attention blocks
    Alternating InnerTrans (local self-attn) + OuterTrans (global cross-attn) stacked layers.

Result: End-to-end 10k modeling at ~50% FLOPs of a vanilla Transformer.

Methodology

① JAGUAR — Training Efficiency

  • Mixed precision: BF16 forward + FP32 gradient accumulation
  • Activation recomputation: gradient checkpointing saves GPU memory
  • Async embedding I/O: CPU lookup overlapped with GPU compute
  • Outcome: 10k-length training on existing hardware, no extra infra

② KV Cache — Inference Efficiency

  • Offline: user history K/V states pre-computed and cached on disk
  • Online: only candidate-item cross-attention computed at request time
  • Incremental update: new behaviors appended without full recomputation
  • Outcome: significant latency reduction; production-ready serving pipeline

Scaling Laws

First scaling-law study for industrial recommendation (R² ≈ 0.97–0.99).

  • Token length: AUC improves log-linearly, 200 → 10k.
  • Params: power-law between model size and AUC.
  • FLOPs: compute-optimal allocation derivable — analogous to Chinchilla laws.

Key insight: At fixed compute, longer history beats more parameters.

Significance

  • Breaks the two-stage ceiling: first practical end-to-end 10k-item history system in production; eliminates the pre-filtering information bottleneck.
  • Scaling laws for RecSys: power-law scaling holds for length, params, and FLOPs (R²≈0.97–0.99) — analogous to LLM Chinchilla laws.
  • JAGUAR as a recipe: long-sequence training on standard GPU clusters; no specialized hardware; replicable by other teams.
  • Deployed at scale: billions of users on Douyin, Douyin Ads, ByteDance e-commerce with measurable revenue lift.
  • Limitation: KV Cache storage ∝ history size; cold-start and privacy constraints not yet addressed.

Impact on My Research

  • Global tokens: G learnable tokens (G≪N) reduce attention cost O(N²) → O(G·N); useful for stabilizing long-context modeling.
  • Token merge: cosine-similarity-based merging cuts effective sequence length ~50% before any attention layer.
  • Hybrid attention: alternating InnerTrans (local self-attn) + OuterTrans (global cross-attn) for multi-scale sequence modeling.
  • KV Cache serving: pre-cache user K/V offline; only candidate-side attention at serving time — applicable to any user-item cross-attention model.
  • JAGUAR template: BF16 + gradient checkpointing + async embedding I/O as standard long-sequence training recipe.

Summary

LONGER enables end-to-end 10k-sequence modeling with ~50% FLOPs cut; scaling laws validated; deployed at ByteDance, billions of users.
