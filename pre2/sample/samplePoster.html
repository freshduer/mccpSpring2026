Accelerating Graph Similarity Search via Efficient GED Computation

Presenter: [Your Name] · Principal Supervisor: [Supervisor Name]

Department of Mathematics
Layman’s summary We study how to search a large database of networks (graphs) for ones that are “similar” to a query. Similarity is measured by the minimum number of edits needed to turn one graph into another (graph edit distance, GED). We propose a new method that is both faster and uses less memory than the current best method, so it can scale to larger graphs and thresholds.

Introduction

Graph similarity search is fundamental in graph databases. Exact matching often returns too few results; similarity search under a threshold τ is needed. The state-of-the-art AStar-LSa computes GED via vertex mappings and a lower bound lbLSa, but its memory consumption grows quickly with graph size and τ, limiting scalability.

Contribution: We propose a two-stage approach: (1) a tighter lower bound lbBMa (branch match-based); (2) a more efficient variant lbBMao that trades a little tightness for O(n³) complexity. The resulting algorithm AStar-BMao runs faster and uses much less memory than AStar-LSa.

Methodology

  • Baseline: AStar-LSa with anchor-aware label set lower bound lbLSa.
  • Ideal solution (lbBMa): Branch matching instead of label sets; formally tighter than lbLSa; complexity O((|V(q)|+|V(g)|)⁴).
  • Practical solution (lbBMao): Slight relaxation of lbBMa so all children’s lower-bound costs are O((|V(q)|+|V(g)|)³) total; still tighter than lbLSa.

Results

Experiments on real datasets show that AStar-BMao runs faster and consumes much less main memory than AStar-LSa. AStar-BMa (using lbBMa) has the smallest search space but higher time cost; AStar-BMao achieves the best overall trade-off.

Discussion / Conclusion

We address the scalability limitation of AStar-LSa by a two-stage refinement: a tighter branch match-based lower bound (lbBMa) and an efficient variant (lbBMao). The result is both faster and more memory-efficient, enabling graph similarity search at larger scale. Future work may combine with approximate methods or other graph metrics.
